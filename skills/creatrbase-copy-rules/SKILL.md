# creatrbase-copy-rules

## Description
Hard formatting and copy rules for all Creatrbase written output.

## When to use
Every piece of writing. Non-negotiable rules that override style preferences.

## Purpose
Consistency across all output: newsletters, admin UI, editorial, summaries.

## Rules
1. UK English only. "customise" not "customize", "colour" not "color", "analyse" not "analyze".
2. No em-dashes or en-dashes. Never use — or –. Use commas, hyphens, or rewrite.
3. No ellipses in body copy. Acceptable only in quoted speech.
4. No emoji in newsletters. Acceptable in admin UI labels and chat.
5. No ALL CAPS except labels (e.g. "NEWSLETTER", "METHODOLOGY"). Never for emphasis.
6. Numbers under 10 as words ("three sends per week"). 10+ as numerals ("12 items").
7. Currency with symbols, no spaces: "£9.99", "$10", not "GBP 9.99".
8. Times in 24-hour format in admin ("08:00"), 12-hour in public-facing ("8am").
9. Dates as "18 April 2026", never "04/18/2026" or "April 18th, 2026".
10. No "leverage", "unpack", "deep-dive", "ecosystem", "synergy", "holistic".
11. No "read more" or "click here". Use descriptive link text.
12. One space after full stops, never two.
13. Oxford comma: "creator economy, AI, and platform updates".
