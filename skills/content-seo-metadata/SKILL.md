## Description
SEO metadata generation — title, meta description, and slug for any Creatrbase content type.

## Purpose
Generate search-optimised metadata that accurately represents the content and is calibrated for independent creator search intent.

## Title Rules

- 50-60 characters (hard limit: 65 characters)
- Front-load the primary keyword
- No clickbait ("You Won't Believe...", "The Secret to...")
- No trailing brand suffix ("| Creatrbase") — the CMS adds this
- Specific over generic: "Gaming Creator CPM Rates 2026: UK Benchmarks" beats "Gaming Creator Earnings Guide"
- Question format acceptable if it matches clear search intent: "How Much Do Gaming Creators Earn from Brand Deals?"

## Meta Description Rules

- 140-155 characters (hard limit: 160 characters)
- Must contain the primary keyword
- Should answer: what does this page tell you and why does it matter?
- No "Learn how to..." or "Discover..." openers
- End with a concrete benefit or data point where possible
- Not a repeat of the title — complement it

## Slug Rules

- Lowercase, hyphen-separated
- 3-6 words
- Contains primary keyword
- No stop words (a, the, and, for, of, in) unless essential for meaning
- No trailing numbers unless date-specific
- Examples:
  - `gaming-creator-cpm-rates-uk`
  - `brand-deal-negotiation-without-agent`
  - `compare-creatrbase-vs-grin`

## Schema Hints

For blog articles: Article schema
For comparison pages: no specific schema needed
For niche pages: no specific schema needed
For research reports: include estimated reading time and methodology reference

## Common Mistakes

- Stuffing multiple keywords into the title — pick one primary
- Meta description that just restates the title
- Slug that includes years (gets stale) unless the content is explicitly annual data
